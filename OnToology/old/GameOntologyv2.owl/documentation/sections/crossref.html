Achievementc back to ToC or Class ToC

IRI: http://purl.org/net/VideoGameOntology#Achievement

Achievement is a reward gained in game due to some event done in the game. Achievements are a standard of game industry to reward players with accomplishing tasks in the game. Achievement is a sbuclass of GameOffering, as In some situations an achievement could also be purchased inside a game.

The subclasses of Achievement are based on the classification done by Markus Montola.

has sub-classes
Collection c, Completion c, Curiosity c, Fandom c, HardMode c, Loyalty c, Luck c, Minigame c, Multiplayer c, Paragon c, SpecialPlayStyle c, Tutorial c, Veteran c, Virtuosity c
is in domain of
isAchievementInGame op, isReceivedInSession op
is in range of
ownsAchievement op, unlocksAchievement op

ownsCharacterop back to ToC or Object Property ToC

IRI: http://purl.org/net/VideoGameOntology#ownsCharacter

ownsCharacter connects player to a character. A player can have multiple characters in one game and this connection is used to define all the different characters a player could be playing. ownsCharacter describes an ownership of the character. Even though a player deletes, trades or loses his/her character in any way, the relationship is still kept. This is to keep the historical knowledge of a Player. In situation where a character should be marked “removed” from the player, use LostEvent class of InstantaneousEvent class.

has domain
Player c
has range
Character c

ownsItemop back to ToC or Object Property ToC

IRI: http://purl.org/net/VideoGameOntology#ownsItem

ownsItem connects a character to certain item. A character can own multiple items and this is used to keep track of the character owned items. OwnsItem describes an ownership of an item. Even though a player loses an item, the ownsItem connection is still held with the item. This is to keep track of character history. As a player can play multiple characters, a player should not be connected to an item. In situation where an item should be marked “removed” from the character, use LostEvent class of InstantaneousEvent class.

has domain
Character c
has range
Item c
